🚀 Create Job Poster - Quick Setup

⚠️ Important Note:
Firebase Phone Authentication ALWAYS requires OTP verification for security.
This is a Firebase security requirement that cannot be bypassed.

What this tool does:
1. Creates a user account with phone number in Firebase Auth
2. Creates the Firestore document with all required fields
3. Marks the user as verified

To login: You'll still need to receive and enter the OTP code sent to the phone number.
📋 Solution:
Use a real phone number you have access to, so you can receive the OTP.
Or use Firebase Test Phone Numbers (configured in Firebase Console).
Use your real phone number to receive OTP
💡 Alternative: Use Firebase Test Phone Numbers

To bypass OTP completely during development:
1. Go to Firebase Console → Authentication → Sign-in method
2. Click on Phone → Phone numbers for testing
3. Add test phone number: +923001234567
4. Set test code: 123456
5. Now you can login with this number using code 123456 (no real SMS sent)
